# Habr Auto Poster

Telegram-бот для автоматического парсинга статей с Habr.com и публикации их в Telegram-канал с использованием Gemini AI для обработки контента.

## Возможности

- Автоматический мониторинг новых статей на Habr.com
- Обработка контента через Gemini AI
- Публикация статей в Telegram-канал
- Настраиваемые интервалы проверки и ожидания
- Пользовательские промпты для AI
- Поддержка нескольких пользователей с индивидуальными настройками
- Docker-контейнеризация для простого развертывания

## Быстрый старт

### 1. Клонирование репозитория

```bash
git clone https://github.com/yourusername/HabrAutoPost.git
cd HabrAutoPost
```

### 2. Получение Telegram Bot Token

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям: введите имя бота и username
4. BotFather отправит вам токен в формате `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`
5. Сохраните этот токен для следующего шага

### 3. Получение Gemini API Key

1. Перейдите на [Google AI Studio](https://aistudio.google.com/app/apikey)
2. Войдите в свой Google аккаунт
3. Нажмите "Get API Key" или "Create API Key"
4. Скопируйте созданный API ключ
5. Сохраните его для настройки бота

### 4. Узнать ID Telegram-группы

**Способ 1: Через бота**
1. Добавьте бота [@userinfobot](https://t.me/userinfobot) в вашу группу
2. Бот автоматически отправит информацию о группе, включая ID
3. ID группы будет в формате `-1001234567890` (всегда начинается с минуса)

**Способ 2: Через веб-версию**
1. Откройте группу в веб-версии Telegram (web.telegram.org)
2. Посмотрите на URL: `https://web.telegram.org/k/#-1001234567890`
3. Число после `#` и есть ID группы

### 5. Настройка окружения

Создайте файл `.env` в корне проекта:

```env
BOT_TOKEN="ваш_токен_от_BotFather"
SQLALCHEMY_URL="postgresql+asyncpg://berlogavpn:misha1102@localhost/hat"
```

**Важно:** Для Docker используйте `db` вместо `localhost`:
```env
SQLALCHEMY_URL="postgresql+asyncpg://berlogavpn:misha1102@db/hat"
```

### 6. Запуск через Docker (рекомендуется)

```bash
# Сборка и запуск контейнеров
docker-compose up -d --build

# Просмотр логов
docker-compose logs -f bot

# Остановка
docker-compose down

# Остановка с удалением данных БД
docker-compose down -v
```

### 7. Запуск без Docker (локально)

**Требования:**
- Python 3.14+
- PostgreSQL 18+

**Установка зависимостей:**

```bash
pip install -r requirements.txt
```

**Запуск PostgreSQL:**

```bash
# Создайте базу данных
createdb hat

# Или через psql
psql -U postgres
CREATE DATABASE hat;
```

**Запуск бота:**

```bash
cd src
python main.py
```

## Использование

### Первый запуск

1. Найдите вашего бота в Telegram по username
2. Отправьте команду `/start`
3. Нажмите "Настройки"
4. Заполните все необходимые параметры:
   - **API ключ** - ваш Gemini API ключ
   - **Модель** - название модели (по умолчанию `gemini-2.5-flash-lite`)
   - **URL** - адрес RSS-ленты Habr (по умолчанию `ru/feed/`)
   - **ID группы** - ID вашего Telegram-канала (начинается с минуса)
   - **Промпт** - инструкции для AI (опционально)

### Настройка задержек

1. Откройте "Настройки" → "Задержка постинга"
2. Настройте:
   - **Интервал проверки** - как часто проверять новые статьи (минимум 10 секунд)
   - **Время ожидания** - максимальное время ожидания новой статьи (минимум 60 секунд)

### Запуск автопостинга

1. Вернитесь в главное меню
2. Нажмите "Запустить автопост"
3. Бот начнет мониторить Habr и публиковать статьи

### Остановка автопостинга

Отправьте боту слово **"Стоп"** (регистр не важен)

## Технологии

- **Python 3.14** - основной язык
- **Aiogram 3.26** - фреймворк для Telegram-ботов
- **SQLAlchemy 2.0** - ORM для работы с БД
- **PostgreSQL 18** - база данных
- **Google Gemini AI** - обработка контента
- **BeautifulSoup4** - парсинг HTML
- **httpx** - асинхронные HTTP-запросы
- **Docker & Docker Compose** - контейнеризация

## Конфигурация

### Переменные окружения (.env)

- `BOT_TOKEN` - токен Telegram-бота
- `SQLALCHEMY_URL` - строка подключения к PostgreSQL

### Настройки в config.py

- `CHECK_NEW_ARTICLE_INTERVAL_SECONDS` - интервал проверки новых статей (по умолчанию 3600 сек / 1 час)
- `POLL_INTERVAL_SECONDS` - максимальное время ожидания (по умолчанию 7200 сек / 2 часа)
- `SEND_DELAY_SECONDS` - задержка между отправкой частей статьи (по умолчанию 3 сек)
- `RETRY_DELAY_SECONDS` - задержка перед повторной попыткой при ошибке (по умолчанию 60 сек)

## Устранение неполадок

### Бот не запускается

- Проверьте правильность `BOT_TOKEN` в `.env`
- Убедитесь, что PostgreSQL запущен
- Проверьте логи: `docker-compose logs bot`

### Ошибка подключения к БД

- Для Docker используйте `db` вместо `localhost` в `SQLALCHEMY_URL`
- Проверьте, что PostgreSQL контейнер запущен: `docker-compose ps`

### Бот не публикует статьи

- Убедитесь, что бот добавлен в группу как администратор
- Проверьте правильность ID группы (должен начинаться с минуса)
- Проверьте, что Gemini API ключ валиден

### Ошибки Gemini API

- Проверьте квоты API на [Google AI Studio](https://aistudio.google.com/)
- Убедитесь, что API ключ активен
- Попробуйте другую модель (например, `gemini-1.5-flash`)

### Docker проблемы

Если у вас проблемы с доступом к Docker Hub:
```bash
# Используйте зеркала или соберите образ локально
docker build -t habrautopost .
```
